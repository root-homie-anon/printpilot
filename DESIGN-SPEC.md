# PrintPilot — Spreadsheet Design Specification

## Purpose
This document defines the quality standard for Google Sheets products generated by PrintPilot. Any agent working on the renderer (`src/renderer/google-sheets.ts`) or designer (`src/agents/designer.ts`) MUST follow these specifications.

---

## Target Quality: Premium Etsy Spreadsheets ($5-10 range)

The output must match what top-selling Etsy spreadsheet sellers produce. These sellers generate $100k-$280k/year. Their products are NOT basic data grids — they are polished, branded, visually rich digital products.

---

## Architecture Overview

```
AI Prompt (designer.ts) → SheetSpec[] → Google Sheets Renderer (google-sheets.ts) → Google Sheets API
                                      → Excel Renderer (spreadsheet.ts) → .xlsx backup
```

### Key Files
- `src/agents/designer.ts` — AI prompt that generates SheetSpec JSON (function: `buildSpreadsheetPrompt`)
- `src/renderer/google-sheets.ts` — Translates SheetSpec → Google Sheets API calls
- `src/renderer/spreadsheet.ts` — Translates SheetSpec → ExcelJS .xlsx (backup format)
- `src/types/index.ts` — ProductBrief, ProductFormat types

### Data Flow
1. Designer agent sends a prompt to Claude requesting spreadsheet design as JSON
2. Claude returns an array of `SheetSpec` objects (8+ sheets)
3. `renderGoogleSheet()` creates a Google Sheet via API with formatting
4. `renderSpreadsheet()` creates an .xlsx backup via ExcelJS
5. Both URLs/paths are stored in `state/products/{id}/design.json`

## Required Visual Elements

### Dashboard Tab
- [ ] Large title banner (26px+ bold, dark background, full-width merged)
- [ ] Accent color subtitle bar (12px bold, primary color background)
- [ ] Thin accent divider line (6px height row)
- [ ] "KEY METRICS" section header with thick left+bottom accent border
- [ ] KPI cards: label row (9px uppercase, light bg) + value row (28px+ bold, accent color text)
- [ ] KPI cards must have visible borders (SOLID on sides, SOLID_MEDIUM accent on top/bottom)
- [ ] "PROGRESS SUMMARY" section with same header treatment
- [ ] Summary table with dark header row (11px bold white text on primaryDark)
- [ ] Alternating rows using `primaryVeryLight` (accent at 92% lightness) vs white
- [ ] Footer with italic help text
- [ ] Hidden gridlines
- [ ] No frozen rows (dashboard is a display, not data entry)

### Data Tabs (Exercise Log, Meal Planning, etc.)
- [ ] Merged title bar (16px bold, CHARCOAL bg, accent bottom border)
- [ ] Accent stripe below title (10px bold, primary color bg)
- [ ] Column headers (11px bold, primaryDark bg, white text, thick accent bottom border)
- [ ] Frozen header rows (top 3 rows frozen)
- [ ] Alternating row colors (primaryVeryLight / white)
- [ ] All cells have light bottom+right borders
- [ ] Text wrapping enabled on all cells
- [ ] 5-7 pre-filled example rows with realistic data
- [ ] 50+ empty rows for user input (with formula columns pre-filled)
- [ ] Dropdown data validation on all dropdown columns (up to 500 values)
- [ ] Formula columns auto-calculate (calories, BMI, % complete, streaks, etc.)

### Instructions Tab
- [ ] Large title (22px bold, CHARCOAL bg, emoji prefix)
- [ ] Accent subtitle bar
- [ ] 6 numbered steps with:
  - Step number in accent-colored cell (18px bold white)
  - Step title (14px bold, primaryDark color)
  - Step description (11px, slate color, text wrapping ON)
- [ ] Spacer rows between steps
- [ ] Footer with branding
- [ ] Hidden gridlines
- [ ] 3-column layout: narrow number | wide content | narrow margin

## SheetSpec Interface

The AI generates this JSON structure. The renderer MUST support all fields:

```typescript
interface SheetSpec {
  name: string;                              // Tab name
  sheetType?: 'dashboard' | 'data' | 'reference' | 'instructions' | 'settings';
  columns: ColumnSpec[];                     // Column definitions
  rows: Record<string, unknown>[];           // Pre-filled data rows
  mergedHeaders?: MergedHeader[];            // Section grouping headers
  conditionalRules?: ConditionalRule[];      // Traffic-light rules
  formatting?: FormattingSpec;               // Sheet-level formatting
}

interface ColumnSpec {
  header: string;
  key: string;
  width?: number;                            // Multiplied by 10 for pixels
  type?: 'text' | 'number' | 'date' | 'dropdown' | 'formula' | 'currency';
  dropdownValues?: string[];                 // For type='dropdown'
  formula?: string;                          // For type='formula', use {ROW} placeholder
  numberFormat?: string;
}

interface ConditionalRule {
  column: string;                            // Must match a column key
  type: 'traffic-light';
  thresholds?: { green: number; yellow: number };  // MUST be numeric, skip if not
}
```

## AI Prompt Requirements (designer.ts)

The `buildSpreadsheetPrompt()` function must request:
- 8+ sheets minimum (Dashboard + 6-7 data tabs + settings/goals)
- 8-12 columns per data sheet
- 20-30 dropdown options per dropdown column (real values, not placeholders)
- 5-7 example rows with realistic data per sheet
- Cross-sheet formulas on Dashboard (referencing data tabs)
- Calculated fields: totals, averages, % complete, streaks, deltas
- Conditional formatting rules with NUMERIC thresholds only
- The response must be valid JSON (array of SheetSpec objects)

## Charts and Sparklines

### Embedded Charts (via addChart batchUpdate)
The renderer automatically adds charts based on data sheet structure:

1. **Donut Chart** on Dashboard — uses the first data sheet with a dropdown column (categories) and numeric column (values). Positioned as overlay at row 10. `pieHole: 0.4` for donut style.

2. **Column Chart** on first suitable data sheet — uses multiple numeric columns. Positioned below the data rows. Shows bar comparison.

Chart creation is wrapped in try/catch — if it fails, the sheet still works without charts.

### Sparklines (via SPARKLINE formula)
Dashboard includes a "TRENDS" section with inline sparkline formulas:
```
=SPARKLINE('Sheet Name'!C4:C30,{"charttype","line";"color","#4F46E5";"linewidth",2})
```
- Up to 4 sparklines, one per data sheet
- Uses the first numeric column from each data sheet
- Color matches the accent palette

### Chart Data Ranges
Charts reference the data sheets by sheetId and row ranges:
- Start at row 2 (header) through row 3 + dataRows
- Categories from dropdown/text column, values from numeric columns
- headerCount: 1 tells the chart API the first row is labels

## Conditional Formatting Rules

- Only `traffic-light` type is supported
- Thresholds MUST be numeric — skip any rule where `Number(threshold)` is NaN
- Apply to rows 3-100 (not just 50)
- Order matters: green first (>= green), then yellow (>= yellow), then red (< yellow)
- Use Google Sheets API types: `NUMBER_GREATER_THAN_EQ`, `NUMBER_LESS`
- Bold text in conditional formatted cells

## Google Sheets API Auth

Uses OAuth2 (not service account). Files:
- `.credentials/google-oauth-client.json` — OAuth2 client config
- `.credentials/google-oauth-token.json` — Refresh token (auto-refreshes)
- Run `scripts/google-auth.ts` to re-authorize if token expires

Sheets are created in the user's Google Drive and shared as "anyone with link can view".

## MCP Server: google_sheets

Installed via `pip install mcp-google-sheets` (xing5/mcp-google-sheets v0.6.1).
Configured in `.mcp.json`. Provides 19 tools:

- `create_spreadsheet` — create new sheets
- `copy_spreadsheet` — copy existing template
- `batch_update_cells` — write data to ranges
- `add_sheet` / `rename_sheet` / `copy_sheet` — manage tabs
- `create_chart` — add charts (column, bar, line, pie, scatter, combo, histogram)
- `share_spreadsheet` — set permissions
- `list_spreadsheets` / `search_spreadsheets` — find existing sheets
- `read_range` / `write_range` — cell-level I/O

Auth: Uses OAuth2 tokens at `.credentials/google-oauth-client.json` + `.credentials/mcp-token.json`.

**Note**: The MCP server handles data and charts but NOT cell formatting (colors, fonts, borders). Formatting still requires direct Google Sheets API batchUpdate calls.

## Common Pitfalls

1. **Non-numeric conditional thresholds**: AI sometimes generates `"High"` instead of `80`. Always validate with `Number()` and skip NaN.
2. **Google Sheets API enum values**: Use `NUMBER_LESS` not `NUMBER_LESS_THAN`. Use `NUMBER_GREATER_THAN_EQ` not `NUMBER_GREATER_THAN_OR_EQUAL`.
3. **Streaming required for large tokens**: When `maxTokens > 16384`, use `client.messages.stream()` in claude.ts.
4. **Dropdown limit**: Google Sheets allows up to 500 values in ONE_OF_LIST validation.
5. **Font**: Default to 'Inter' font family. Falls back to sans-serif.
6. **Merges before data**: Create spreadsheet with data first, then apply merges via batchUpdate.
7. **Service account quota**: Service accounts have 0 Drive storage. Always use OAuth2.

## Quality Checklist Before Shipping

- [ ] Open the Google Sheet link — does it look professional at first glance?
- [ ] Dashboard: Are KPI numbers large and prominent? Do they stand out?
- [ ] Data tabs: Are dropdowns working? Click a cell in a dropdown column.
- [ ] Formulas: Enter test data — do calculated columns update?
- [ ] Headers: Are they frozen? Scroll down — do headers stay visible?
- [ ] Colors: Is the accent color consistent across all tabs?
- [ ] Alternating rows: Clear visual distinction between rows?
- [ ] Text: No cut-off text? Wrapping works on long content?
- [ ] Instructions: Clear, numbered, professional?
- [ ] Compare against top Etsy listings in the niche. Does it hold up?
